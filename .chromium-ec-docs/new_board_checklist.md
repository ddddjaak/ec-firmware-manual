If you&#39;re creating a new board based on existing baseboard, you can jump straight to the relevant link found under <a href="#Configure-EC-Features">Configuring EC Features</a> and focus on known board changes.</p><h2><a class="h" name="Conventions" href="#Conventions"><span></span></a><a class="h" name="conventions" href="#conventions"><span></span></a>Conventions</h2><h3><a class="h" name="Key-Files" href="#Key-Files"><span></span></a><a class="h" name="key-files" href="#key-files"><span></span></a>Key Files</h3><p>Before you get started, it&#39;s important to understand the role of a few key files in the EC codebase.</p><ul><li><p><a href="/chromiumos/platform/ec.git/+/HEAD/include/config.h"><code class="code">include/config.h</code></a> {#config_h} - Contains the list of top-level configuration options for the Chrome EC codebase. Each configuration option is documented inline and is considered the authoritative definition.</p></li><li><p><code class="code">baseboard/&lt;name&gt;/</code> - This directory contains header files and source files shared by all boards in a baseboard family.</p><ul><li><code class="code">baseboard.h</code> - Contains the EC configuration options shared by all devices in the baseboard family.</li><li><code class="code">baseboard.c</code> - Contains code shared by all devices in the baseboard family.</li><li><code class="code">build.mk</code> - The board family makefile specifies C source files compiled into all boards in the baseboard family.</li></ul></li><li><p><code class="code">board/&lt;board&gt;</code> - Files in this directory are only built for a single board.</p><ul><li><code class="code">board.h</code> - EC configuration options specific to a single board.</li><li><code class="code">board.c</code> - Code built only on this board.</li><li><code class="code">build.mk</code> {#board_build_mk} - The board makefile defines the EC chipset family, defines the baseboard name, and specifies the C source files that are compiled.</li><li><code class="code">gpio.inc</code> - This C header file defines the interrupts, GPIOs, and alternate function selection for all pins on the EC chipset.</li><li><code class="code">ec.tasklist</code> - This C header defines the lists of tasks that are enabled on the board. See the main EC documentation more details on <a href="/chromiumos/platform/ec.git/+/HEAD/README.md#Tasks">EC tasks</a>.</li></ul></li></ul><h3><a class="h" name="GPIO-Naming" href="#GPIO-Naming"><span></span></a><a class="h" name="gpio-naming" href="#gpio-naming"><span></span></a>GPIO Naming</h3><p>Many drivers and libraries in the common EC code rely on board variants defining an exact GPIO signal name. Examples include the <code class="code">GPIO_LID_OPEN</code>, <code class="code">GPIO_ENTERING_RW</code>, and <code class="code">GPIO_SYS_RESET_L</code> signals. The net names in schematics often do not match these names exactly.
