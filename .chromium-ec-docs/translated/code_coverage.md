# 代码覆盖率

概述如何在 EC 代码库中运行单元测试时使用代码覆盖率工具。

## 测试覆盖率要求

所有对 EC 代码的修改，都要求新增或修改的代码行至少达到 80% 的测试覆盖率。

## 可用性

代码覆盖率仅适用于宿主机上的单元测试，不适用于在目标硬件上运行的手动测试。

## 构建代码覆盖率

要构建带代码覆盖率的宿主机单元测试，使用 `coverage` 目标运行 `make`：

```bash
make coverage -j
```

该目标会编译并链接带有 `--coverage` 标志的单元测试（引入 `gcov` 库），运行测试，然后使用 `lcov` 和 `genhtml` 工具将分析数据生成代码覆盖率报告。

覆盖率报告的顶层页面位于 `build/coverage/coverage_rpt/index.html`。

要获取某个特定板级的覆盖率报告，执行以下命令：

```bash
BOARD=<board_name> make coverage -j
```

### 构建输出中的噪声

构建过程中可能会出现一些警告信息，这些通常不影响最终的覆盖率报告。

## Zephyr ztest 代码覆盖率

对于基于 Zephyr 的测试，代码覆盖率的收集方式有所不同。使用 twister 运行测试时，可以通过参数启用覆盖率收集。

## CI 中的代码覆盖率

### 代码搜索

覆盖率报告可以通过 Chromium 代码搜索平台查看。

### 提交前检查

在代码提交前（Presubmit），CI 系统会自动运行测试并检查覆盖率是否达标。

### 提交后检查

代码合入后（Post-submit），CI 系统会生成完整的覆盖率报告供团队审查。
