# 创建新的 Zephyr EC 项目

## 概述

本文档描述为 Chromebook 设计创建新的 Zephyr EC 项目所需的高级步骤。

## 目标读者

本文档面向在 EC 代码库中工作的软件工程师。本文档及相关链接文档为创建新的 Zephyr EC 项目以及修改/维护现有 Zephyr EC 项目提供指南。

## 如何使用本文档

以下各节详细介绍了需要为板级设计修改的单一功能集。每个功能集可以通过合理大小的变更列表实现，并且可以独立工作。

每个功能包含以下子任务：

- **Kconfig 选项** — 与该功能相关的 Kconfig 选项。Kconfig 选项在项目配置文件中启用。
- **设备树节点** — 该功能所需的设备树（Devicetree）节点和属性。
- **板级特定代码** — 项目必须实现的 C 代码。
- **线程** — 该功能创建的线程及其概述。
- **测试与调试** — 测试 EC 功能集和调试问题的策略，以及相关的 EC 控制台命令。
- **示例** — 基于现有板级实现配置 EC 功能的完整示例。

## 向 zmake 添加新项目

参考项目配置文档将新的板级项目添加到 zmake。

### USB-C 端口

Zephyr EC 应用支持两种不同的技术栈来支持 USB Type-C 端口和 USB PD（Power Delivery）：

1. **TCPMv2**：2024 年之前创建的项目使用 TCPMv2（Type-C Port Manager version 2）技术栈。
2. **PDC**：2024 年及之后创建的项目使用 Power Delivery Controller（PDC）芯片支持 USB Type-C 端口。
