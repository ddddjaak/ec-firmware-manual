## 调用链路

```text
app 线程 ──k_sleep()/k_sem_take()/device_ioctl(dev)──▶ Zephyr 内核调度
                                                       │
                                                       ▼
                                              设备驱动 .ioctl() 回调
                                                       │
                                                       ▼
                                              HAL gpio_pin_set() / i2c_transfer()
                                                       │
                                                       ▼
                                              芯片寄存器操作
```

## 各层详解

### ① EC 应用层 — `app/`

15 个功能模块，每个由独立 Zephyr 线程驱动（`K_THREAD_DEFINE`），周期性运行。

| 模块 | 线程周期 | 核心职责 |
|------|----------|----------|
| `power_management` | 1ms | 电源状态机（S0/S3/S5）、休眠唤醒 |
| `thermal_management` | 100ms | 温度采集、风扇策略、过热保护 |
| `battery` / `charge` | 100ms | 电量计算、充电曲线、电池保护 |
| `pd_management` | 事件驱动 | USB PD 协议栈、角色协商、Alt Mode |
| `kbchost` / `smchost` | 1ms | KBC 键码转发 / SMC 主机通信 |
| `peripheral_management` | 1ms | 外设模块调度中枢（`periph_thread`） |
| `power_sequencing` | 1ms | 上电时序控制（`pwrseq_thread`） |
| `debug` | 按需 | Shell 命令、诊断转储、RTT 日志 |

> 每个模块**直接调用 Zephyr API**（`k_sleep`、`k_sem_take`、`device_ioctl` 等），不经过中间封装。

### ② EC 驱动层 — `drivers/` + `misc/` + `lib/`

18 个 EC 专用驱动，基于 Zephyr 设备驱动模型（`struct device`），以 `DEVICE_DT_DEFINE` 注册到设备树。

| 类别 | 驱动 | 说明 |
|------|------|------|
| 主机接口 | `eSPI`、`ACPI`、`Port80`、`PECI` | EC 与 Host CPU 的通信桥梁 |
| 输入设备 | `Keyboard (KSCAN)`、`PS2` | 键盘矩阵扫描、PS/2 鼠标键盘 |
| 散热控制 | `Fan`、`MPS (Multi-Phase)` | 风扇 PWM 调速、多相电源控制 |
| 存储与升级 | `Flash (SPI)`、`OTA` | 外部 SPI Flash、固件 OTA 升级 |
| 总线管理 | `I2C`、`GPIO` | I2C 外设管理、GPIO 扩展 |
| 人机交互 | `LED`、`USB` | 指示灯控制、USB 设备模式 |
| 存储外设 | `EEPROM`、`VCI`、`ERPMC` | EEPROM 读写、VCI 接口、ERPMC 安全 |

### ③ Zephyr RTOS 平台 — `ecfwwork/zephyr_fork/`

Zephyr LTS 内核（fork），提供 EC 固件的运行时基础：

- **内核** (`kernel/`)：协作式线程调度、互斥锁、信号量、消息队列、工作队列、定时器
- **子系统** (`subsys/`)：Logging（分级日志）、Shell（UART 命令行）、PM（电源管理）、Storage（Flash Map / 设置存储）
- **设备模型**：`struct device` + `DEVICE_DT_DEFINE` 统一驱动注册框架，主循环 `device_ioctl_traversal` 遍历调度

### ④ 芯海芯片 HAL + 片上外设 — `ecfwwork/zephyr_modules_csce250x/`

芯片 SDK 模块，独立于 Zephyr 源码：

| 组件 | 路径 | 内容 |
|------|------|------|
| HAL | `hal/csce250x/` | CMSIS-Core、标准外设库、ROM 库 |
| SoC | `soc/arm/` | ARM Cortex-M33/M0+ 启动代码、初始化 |
| 片上驱动 | `drivers/` (25 个) | GPIO、I2C、SPI、UART、ADC、PWM、DMA、eSPI、PECI、KSC、Flash、RTC、WDT 等 |
| 板级 | `boards/` | 评估板 DTS、pinmux、Kconfig |

### ⑤ 硬件

| 芯片 | 内核 | 定位 |
|------|------|------|
| **CSCE2010** | Cortex-M0+ | 轻量 EC，低功耗场景 |
| **CSCE250X** | Cortex-M33 | 高性能 EC，主推平台 |
| **CSCE2520** | Cortex-M33 + TrustZone | 新一代 EC，安全增强 |

> 各模块的详细功能与使用方法见 [固件功能模块说明](/ec-firmware-manual/docs/development/modules/01_system_infrastructure)。
