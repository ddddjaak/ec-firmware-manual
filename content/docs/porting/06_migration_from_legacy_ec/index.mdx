## 国产替代：EC 芯片的供应链重构

过去二十年，全球 EC（嵌入式控制器）芯片市场长期由两家台系厂商主导——**ITE（联阳半导体）** 和 **Nuvoton（新唐科技）**。从 Chromebook 到 ThinkPad，从消费笔记本到工业一体机，几乎所有 x86 计算设备的 EC 固件都运行在这两家芯片上。

然而，三个宏观趋势正在重塑这一格局：

**1. 供应链安全诉求**

地缘政治紧张使半导体供应链从"全球分工最优"转向"区域安全可控"。对于国内笔记本 OEM 厂商而言，EC 芯片虽小，却是主板上的**刚需物料**——没有 EC，整机无法上电。一颗 3 美元的芯片可以卡住一台 500 美元笔记本的出货。在芯片出口管制常态化的背景下，"可替代性"本身就是供应链价值。

**2. 技术架构的代际更替**

传统 EC 芯片长期停留在 8 位 C51 内核（ITE）或 Cortex-M4（Nuvoton），软件开发模式仍以裸机 + 寄存器宏为主。这与当前 MCU 行业的主流趋势——**RTOS 标准化 + 设备树分离硬件描述 + 统一驱动模型**——产生了代际差距。Zephyr RTOS 作为 Linux 基金会旗下的开源嵌入式平台，正在成为行业标准。芯海 EC 方案从第一天就基于 Zephyr 设计，不是"移植 Zephyr 到老芯片"，而是"为 Zephyr 设计新芯片"。

**3. 成本与供应弹性**

单一供应商依赖意味着议价权丧失和交付风险。引入第二供应商（Second Source）是硬件行业的通行做法，但 EC 芯片的特殊性在于——更换 EC 意味着**固件需要全部重写**，切换成本远高于换一颗 LDO 或 MOSFET。芯海通过提供与 Zephyr 生态完全对齐的开发包 + 完整的迁移工具链，将这一切换成本从"不可能"降为"可规划"。

## 迁移的本质：不仅是换芯片

芯片替代不是简单的 pin-to-pin 替换（EC 芯片也没有标准引脚），而是一次**架构升级**：

```
旧平台：裸机 C51/M4 + 寄存器宏 + 私有构建系统
        ↓ 迁移
新平台：Zephyr RTOS + 设备树 + 驱动模型 + CMake/west
```

迁移的价值不仅是"换个国产芯片"，更是：
- **开发效率**：从寄存器级编程升级到驱动 API 调用
- **代码复用**：Zephyr 生态内跨平台代码直接复用
- **可维护性**：硬件描述从代码中分离到设备树，换板只需改 DTS
- **调试能力**：从 GPIO 翻转 + 逻辑分析仪升级到 Shell + Logging + SystemView
- **安全基线**：获得 MCUboot Secure Boot、TrustZone 隔离等现代安全能力

## 三平台架构对照

不同源平台的架构差异决定了迁移策略的差异。

| 维度 | ITE IT557x | Nuvoton NPCX | Chipsea CSCE250X |
|------|-----------|-------------|-------------------|
| CPU 内核 | 80390 (C51 兼容), 8 位 | ARM Cortex-M4, 32 位 | ARM Cortex-M33, 32 位 |
| 频率 | 48–96 MHz | 48–100 MHz | 120 MHz |
| 地址空间 | 哈佛架构, CODE/XDATA/DATA | Von Neumann, 统一线性 | Von Neumann, 统一线性 |
| RAM | 4–16 KB XDATA + 内部 SRAM | 64–256 KB SRAM | 256+ KB SRAM |
| Flash | Bank 切换 (4×32KB) | 512 KB–1 MB 统一 Flash | 512 KB 统一 Flash |
| RTOS | 裸机 / 服务标志位分发 | 裸机 / FreeRTOS（部分） | Zephyr RTOS |
| 外设访问 | 直接寄存器宏 | BSP HAL 库 / 直接寄存器 | 设备树 + 标准驱动 API |
| 硬件描述 | 代码中硬编码 (OEM_GPIO.H) | BSP 头文件中宏定义 | 设备树 (DTS) 独立文件 |
| 构建系统 | Keil C51 + NMAKE | GCC + Makefile / IAR EWARM | CMake + west |

## 迁移路径选择

<div className="grid-cards my-4">

- **从 ITE IT557x 迁移**

  C51 8 位 → ARM 32 位，架构差异最大。Bank 切换、哈佛内存模型、服务标志位分发全部需要重构。但 ITE 在国内笔记本市场的保有量最大，是国产替代的核心场景。

  [ITE 迁移指南 →](/ec-firmware-manual/docs/porting/06_migration_from_legacy_ec/01_ite)

- **从 Nuvoton NPCX 迁移**

  同为 ARM 架构，内存模型一致。核心差异在于 BSP HAL → Zephyr 驱动模型的转换和外设配置方式（寄存器宏 → DTS）。NPCX 在 Chromebook 生态中广泛使用，迁移价值高。

## 通用迁移原则

无论从哪个平台迁移，以下原则始终适用：

1. **先跑起来再完善**：优先实现最小启动（UART 有输出），再逐步添加外设
2. **GPIO 对齐先行**：在接入任何外设之前，先完成所有引脚的逐一对齐验证
3. **保留原机做参考**：在迁移验证阶段，保留一台运行原固件的机器作为行为对照
4. **逻辑分析仪是必备工具**：电源序列、总线通信等时序行为必须用仪器验证
5. **分阶段交付**：Phase 1 最小启动 → 2 GPIO → 3 总线 → 4 电源 → 5 输入设备 → 6 完整集成

> 详细的通用分阶段策略参见 [新平台移植流程](/ec-firmware-manual/docs/porting/01_porting_guide)。
