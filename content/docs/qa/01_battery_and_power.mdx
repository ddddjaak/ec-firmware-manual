---
title: 电池与电源调试
---

本章节涵盖电池充放电、电源管理、低功耗模式相关的常见调试问题。

## 充放电状态异常

**Q：** 电池电量显示不准（Fuel Gauge 偏差）

**症状**：系统显示电量 50%，实际已耗尽的电池或已充满。

**Triage Checklist**：

1. **复现** — 确认偏差是持续存在还是偶发；记录充电曲线数据
2. **定位** — 检查 Fuel Gauge IC 的 I2C 通信是否正常，读寄存器值是否更新
3. **缩小** — 对比裸机读取 vs Zephyr 驱动读取，排除驱动层问题
4. **根因** — 常见原因：
   - Fuel Gauge 未完成学习周期（首次使用需 2-3 次完整充放电）
   - 电池容量寄存器未校准
   - I2C 读取出错返回旧缓存值
5. **防护** — 添加电量读取超时检测 + 异常值过滤

**Q：** 充电不启动 / 充电电流异常

**症状**：适配器已插入，系统不进入充电状态，或充电电流远低于预期。

**排查思路**：

1. 检查 ACPI 适配器检测通知是否正确发送
2. 检查充电策略是否被热保护或电池保护限制
3. 检查 PWM 充电控制器的配置参数
4. 用逻辑分析仪抓 I2C (SMBus) 确认 charger IC 寄存器写入值

**Q：** 低功耗模式下被异常唤醒

**症状**：系统进入 S3/S5 后立即或随机被唤醒。

**排查思路**：

1. 检查 GPIO 唤醒源配置 — 是否有多余的 GPIO 使能了唤醒
2. 检查 RTC 闹钟是否意外触发
3. 用示波器抓唤醒引脚，确认是哪个事件触发
4. 检查 `CONFIG_PM_DEBUG` 内核日志

## 电源管理 (D-State / S-State)

**Q：** EC 无法正常进入休眠模式

**症状**：系统请求 S3/S5 后，EC 未进入低功耗状态。

**Triage Checklist**：

1. **定位** — 检查 Zephyr PM 框架是否使能 (`CONFIG_PM`)
2. 确认所有设备驱动都正确实现了 `PM_DEVICE` 回调
3. 检查是否有设备因 `pm_device_runtime_get/put` 引用计数导致无法 suspend
4. 使用 `poweroff` / `suspend-to-idle` shell 命令手动测试

**Q：** ACPI 通知未到达 OS

**症状**：EC 侧状态变化（如 AC 插入）后，OS 侧未响应。

**排查思路**：

1. 确认 eSPI/LPC 通道正常（检查 IO 基址和中断线）
2. 检查 EC 的 ACPI 事件队列是否未满或未正确触发 SCI#
3. 在 Linux 侧用 `acpi_listen` 确认 OS 是否收到事件
