---
title: 外设与输入调试
---

本章节涵盖键盘矩阵、触摸板、传感器等外设相关调试问题。

## 键盘矩阵

**Q：** 按键无响应 / Ghost Key

**症状**：单个键不响应，或多个键按下时出现幽灵按键。

**Triage Checklist**：

1. **复现** — 确认是特定键还是整行/列不响应
2. **定位** — 用万用表短路 KSx/KIx 引脚确认硬件通路正常
3. **缩小** — 检查键盘矩阵扫描代码：
   - KS (Keyboard Scan) 输出序列是否正确
   - KI (Keyboard In) 读取时序是否正确
   - 上拉/下拉配置是否匹配硬件设计
4. **根因** — 常见原因：
   - Device Tree 中 `row-size` / `col-size` 配置错误
   - GPIO 引脚被其他功能占用
   - 未加防 ghosting 逻辑（如 3-key rollover 保护）
5. **防护** — 添加按键消抖 + ghost key 检测算法

**Q：** 键盘扫描频率异常

**症状**：按键响应延迟过高或 CPU 占用过高。

**排查思路**：

1. 检查扫描间隔配置（通常 8-16ms）
2. 确认是否使用了中断模式而非轮询模式
3. 用 `kernel stacks` 查看键盘线程栈使用情况


## 触摸板 (I2C-HID)

**Q：** 触摸板无法识别 / 无响应

**症状**：设备管理器中不显示触摸板，或触摸板无反应。

**排查思路**：

1. I2C 扫描确认设备地址应答
2. 检查 HID Descriptor 读取是否正确（地址 0x0001 读 2 字节获取 descriptor 长度）
3. 检查 I2C-HID 协议版本兼容性（v1 vs v2）
4. 检查 GPIO 中断线是否正常（RST# / INT# 引脚状态）

**Q：** 触摸板光标跳动 / 漂移

**症状**：静止时指针自动漂移，或点击位置偏移。

**排查思路**：

1. 检查触摸板固件的 noise threshold 配置
2. 确认电源噪声是否影响触摸检测
3. 检查跳线/排线连接是否松动
4. 更新触摸板固件（如该型号支持）


## 传感器

**Q：** 温度传感器读数异常

**症状**：温度显示 0°C、极高值或跳变。

**Triage Checklist**：

1. **定位** — 确认是单路传感器还是全部传感器异常
2. 检查 ADC 通道配置（如有模拟温度传感器）
3. 检查 I2C/OneWire 通信（如有数字温度传感器）
4. 验证温度计算公式和系数是否正确
5. **常见根因**：ADC 参考电压配置错误 / 传感器驱动未完成校准

**Q：** 加速度传感器方向错误

**症状**：屏幕旋转方向与实际姿态不匹配。

**排查思路**：

1. 检查传感器的物理摆放方向（X/Y/Z 轴对应关系）
2. 在 Device Tree 中配置 `mount-matrix` 进行坐标系转换
3. 用 Shell 命令读取原始数据验证传感器正常工作


## 调试工具速查

| 工具 | 用途 | 命令示例 |
|------|------|----------|
| `evtest` | Linux 输入事件 | `evtest /dev/input/event3` |
| `i2cdetect` | I2C 设备扫描 | `i2cdetect -y -r 0` |
| Zephyr Shell | 传感器数据读取 | `sensor get` |
| 万用表 | GPI O 电平 | 确认 KS/KI 引脚状态 |
