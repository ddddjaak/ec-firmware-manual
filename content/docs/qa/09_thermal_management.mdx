本章节涵盖风扇控制、温度传感器、热保护策略相关的调试问题。

## 风扇控制

**Q：风扇不转或转速异常**

**症状**：系统温度升高但风扇不启动，或风扇全速运转无法降速。

**Triage Checklist**：

1. **复现** — 确认是特定温度区间还是所有状态
2. **定位** — 检查风扇控制链路：
   - PWM 输出引脚是否有波形（示波器）
   - 风扇供电是否正常
   - 风扇转速反馈（TACH）信号是否有脉冲
3. **缩小** — 手动设定 PWM duty cycle 测试：
   ```bash
   # Zephyr Shell 中手动控制风扇
   fan set 80    # 设定 80% duty
   ```
4. **根因** — 常见原因：
   - PWM 控制器未初始化或频率配置错误
   - 风扇控制表 (fan table) 阈值配置不合理
   - TACH 引脚中断未正确使能（转速读取为 0）
   - 风扇电源未使能（GPIO EN 引脚未拉高）
5. **防护** — 添加风扇堵转检测 + 异常转速告警

**Q：风扇转速跳变 / 抖动**

**症状**：风扇转速在相邻两个档位之间频繁切换，产生噪音。

**排查思路**：

1. 检查温度传感器的采样噪声 — 是否因噪声导致跨阈值误判
2. 确认风扇控制是否加了迟滞 (hysteresis)：
   - 升温阈值 60°C → 进入高速，但降温需低于 55°C 才降速
3. 检查是否使用了温度变化率 (dT/dt) 预测代替即时温度
4. 确认 PWM 分辨率是否足够（通常需 8-bit 以上）

## 温度监控

**Q：温度 trip point 未触发 OS 响应**

**症状**：EC 检测到过温，但 OS 未降频或未关机。

**Triage Checklist**：

1. **定位** — 确认链路：
   - EC 侧：温度读数是否超过 trip point 阈值
   - ACPI 通知：`\_TZ` 下的 `_CRT` / `_PSV` 是否被正确评估
   - OS 侧：`acpi -t` (Linux) 或 `!tzinfo` (Windows kernel debugger)
2. **根因** — 常见原因：
   - ACPI Thermal Zone 定义与 EC 实际传感器编号不匹配
   - SCI# 通知未送达（检查中断线）
   - OS 侧 DPTF (Dynamic Platform Thermal Framework) 未启用或策略被覆盖
3. **防护** — 在 EC 侧加硬件过温关机作为最后防线

**Q：多温度源投票策略异常**

**症状**：风扇转速由最高温度决定，但某传感器读数异常导致误触发。

**排查思路**：

1. 检查每个温度传感器的原始读数 — 确认是否有传感器故障（开路/短路）
2. 确认投票逻辑是否正确取最大值：
   ```c
   // 常见模式
   int max_temp = 0;
   for (int i = 0; i < NUM_SENSORS; i++) {
       max_temp = MAX(max_temp, sensor_readings[i]);
   }
   ```
3. 检查是否有传感器被豁免（如 VRM 区域允许更高温度）
4. 考虑加权投票：根据传感器位置赋予不同权重

## 热保护

**Q：过温保护误触发**

**症状**：正常负载下系统频繁进入热保护（降频/关机）。

**排查思路**：

1. 确认温度传感器的校准是否准确（用红外测温仪对比实际芯片温度）
2. 检查 ADC 参考电压和分压系数是否匹配实际电路
3. 确认传感器安装位置是否靠近真实热源（PCB 布局可能导致读数偏差）
4. 检查软件中的温度补偿/偏移量配置

## 调试工具速查

| 工具 | 用途 | 命令示例 |
|------|------|----------|
| Zephyr Shell `fan` | 风扇手动控制 | `fan set 50` / `fan info` |
| Zephyr Shell `sensor` | 温读传感器读取 | `sensor get temp_cpu` |
| 示波器 | PWM/TACH 波形 | 测量 PWM 占空比 / TACH 频率 |
| 红外测温仪 | 实测芯片温度 | 与 EC 读数对比 |
| Linux `acpi -t` | OS 侧热状态 | 查看 Thermal Zone 温度/trip point |
