### 配置体系简介

开发包使用 Zephyr 标准的 Kconfig + 设备树 双轨配置：

- Kconfig：控制功能开关（软件层面）—— 决定哪些模块参与编译；
- 设备树（DTS）：描述硬件连接（硬件层面）—— 决定外设的地址、引脚、速率等参数。

日常开发主要通过 `prj.conf` 启用/关闭功能模块，通过板级 `.dts` 文件调整硬件参数。Kconfig 配置语法、优先级机制与自定义方法详见第5章 [应用开发指南 → Kconfig 配置系统](../appdev/index.md)。
